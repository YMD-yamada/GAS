# 【無料・初心者向け】毎日の「帰宅時間・食事」の連絡を LINE に自動化（GAS × LINE）

**この記事でわかること**

- プログラムが初めてでも、「何をどこで押せばいいか」の流れがわかる
- 家族に毎日送っている連絡を、Google の仕組みと LINE で半自動〜自動に近づける考え方がわかる
- GitHub に載せるときに **絶対に載せてはいけないもの** がわかる

---

## はじめに：なぜ作ったか

毎日、家族（うちでは妻）に「今日の帰宅の目安」と「食事するかどうか」などを連絡していました。  
内容は決まっているのに、毎回手入力するのは手間なので、**決まった情報を LINE に送る仕組み**を Google Apps Script（GAS）と LINE の公式機能（Messaging API）でつくりました。

個人で使う分には **無料枠の範囲で十分** でした。

※ この記事は「仕組みの説明」と「安全に公開するための注意」が中心です。コードそのものは GitHub を参照してください（秘密情報は載せていません）。

**より丁寧な手順（LINE の画面操作から GitHub・note・YouTube まで）**は、同じリポジトリ内の **[setup-beginner.md](https://github.com/YMD-yamada/GAS/blob/main/docs/setup-beginner.md)** を参照してください。

## 用語をかんたんに

| 言葉 | イメージ |
|------|----------|
| **GAS（ジーエーエス）** | Google が無料で提供している「小さなプログラムを動かす場所」。スプレッドシートとセットで使うイメージでも近いです。 |
| **LINE Developers** | LINE の「開発者向けの管理画面」。ここで「メッセージを送るための鍵（トークン）」をもらいます。 |
| **Messaging API** | プログラムから LINE にメッセージを送るための公式の窓口です。 |
| **トークン** | パスワードのようなもの。**誰かに知られると、その人があなたの代わりにメッセージを送れる**ので、絶対に公開しません。 |

---

## 全体の仕組み（30 秒で理解）

1. Google 側（GAS）で「送る文面」を組み立てる  
2. LINE が発行した **トークン** を使って、LINE のサーバーに「この内容を送って」とお願いする  
3. 家族の LINE に通知が届く  

「毎日同じアプリで打っている文章」を、**決まった時間にボタン一つや自動実行で送る**イメージです。

---

## 準備するもの

- Google アカウント（Gmail など）
- LINE アプリが入ったスマホ（家族側・自分側）
- **根気（初回だけ画面が多いです）**

プログラムの書き方を覚える必要はありません。**画面の指示どおりに進められるか**がポイントです。

## 大まかな手順（チェックリスト）

### 1. LINE Developers で「Messaging API」のチャネルを作る

1. [LINE Developers](https://developers.line.biz/) にログイン  
2. プロバイダー・チャネルを作成（画面の「作成」「次へ」に従う）  
3. **チャネルアクセストークン（長期）** を発行できる状態にする  

ここで出てくる **長い文字列がトークン** です。メモ帳にコピーする場合は、**後で公開しない場所**だけに保存してください。

### 2. 公式アカウントを友だちに追加する

通知を受け取る家族が、**あなたが作った公式アカウントを友だち追加**している必要があります。  
初めての方は「友だち追加用の QR コード」を Developers の画面から確認できます。

### 3. Google Apps Script でプロジェクトを作る

1. [script.google.com](https://script.google.com/) を開く  
2. 新しいプロジェクトを作成  
3. GitHub にあるコードを、説明どおりにコピー＆貼り付け（または clasp で同期）

### 4. スプレッドシートに「今日の内容」を書く欄を作る（このリポジトリのコード向け）

1 枚目のシートで、**B1** に帰宅予定、**B2** に食事、**B3** にメモ（任意）を書きます。  
A 列には「帰宅予定」「食事」「メモ」などラベルを書いておくと見やすいです（送信に使うのは主に B 列です）。

詳細は GitHub の **[spreadsheet-layout.md](https://github.com/YMD-yamada/GAS/blob/main/docs/spreadsheet-layout.md)** を参照。

### 5. 秘密情報は「スクリプトのプロパティ」に入れる（重要）

GAS のエディタで **プロジェクトの設定（歯車アイコン）→ スクリプトのプロパティ** に、次のようなものを登録します。名前は **[script-properties.md](https://github.com/YMD-yamada/GAS/blob/main/docs/script-properties.md)** と一致させてください。

- `LINE_CHANNEL_ACCESS_TOKEN`（必須）  
- `LINE_TO_USER_ID`（必須。Webhook で自動取得できることもあります）  
- その他、Webhook 用の `LINE_CHANNEL_SECRET` など  

**コードの中に直接トークンを書かない**のが鉄則です。  
GitHub に上げるとき、うっかりコードにトークンが残っていないか必ず確認してください。

### 6. 動作確認

- まず関数 **`sendTestMessage`** をエディタから実行し、テスト文が届くか確認  
- 次に **`sendFamilyUpdate`** で、スプレッドシートの内容が反映された文が届くか確認  

### 7. 毎日動かす（任意）

「時間主導型のトリガー」で、決まった時刻に自動実行する設定ができます。  
通勤パターンに合わせて調整してください。

## GitHub に公開するときのルール（これだけは守ってください）

**載せてよいもの**

- 処理の流れがわかるコード（トークンや ID が無い状態）  
- この README や解説用の文章  

**載せてはいけないもの**

- チャネルアクセストークン  
- チャネルシークレット  
- 個人を特定できるログや実データ（必要なら匿名化）  

もし誤ってトークンを載せてしまったら、**LINE Developers でトークンを再発行**し、古いものを無効にしてください。

---

## よくあるつまずき

- **メッセージが届かない**  
  - 友だち追加できているか  
  - トークンを間違えてコピーしていないか（前後に空白が入っていないか）  
  - 送信先 ID が正しいか  

- **実行するとエラーになる**  
  - GAS の「実行ログ」を開き、赤い文字のメッセージを検索エンジンで調べるとヒントが出ることが多いです
