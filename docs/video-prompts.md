# 動画作成用プロンプト（ショート動画・15 分動画）

動画の制作は現状予定がなくても、そのまま **生成 AI（台本・構成）** や **編集ソフトのメモ** として使えます。  
読み手は **プログラム未経験の初心者**。トーンは **やさしく、専門用語は言ったあと一言で言い換える**。

---

## 共通ルール（画面収録時）

- **トークン・シークレット・ユーザー ID は画面に映さない**（ダミー文字列またはモザイク）
- 個人が特定される名前・住所・電話は使わない（「家族」「配偶者」などに置き換え）
- 実データのスプレッドシートは見せず、ダミー行のみ

---

## A. ショート動画用（60 秒前後・縦型想定）

### A-1. 目的（何の動画か）

毎日の「帰宅時間・食事の有無」の連絡を、Google と LINE の仕組みでラクにする紹介。

### A-2. 台本プロンプト（生成 AI にそのまま貼れる例）

以下をコピーして、動画台本生成用の AI に貼り付けてください。

```
あなたは日本語のショート動画台本ライターです。縦型ショート（60秒前後）用の台本を書いてください。

条件:
- 視聴者はプログラム未経験者
- 内容: Google Apps Script と LINE Messaging API で、家族に「帰宅の目安」と「食事の有無」をLINE通知する仕組みの紹介
- トーン: やさしい、早口にしすぎない、専門用語は直後に一言説明
- 構成: フック(3秒) → 悩み共感(10秒) → 解決のイメージ(20秒) → ざっくり2ステップ(Google側・LINE側)(20秒) → 注意「トークンは公開しない」(5秒) → CTA「詳細は概要欄のGitHub/Note」(5秒)
- セリフは「です・ます」調
- 画面に出すテロップ用に、重要語句を【】でマーク

出力形式:
1. タイトル案を3つ
2. 秒数付き台本（ナレーション全文）
3. テロップ一覧（時系列）
4. 概要欄に書く短文（200字以内）
```

### A-3. 映像の指示プロンプト（B-roll / AI 動画向け・任意）

```
明るいオフィスでも自宅でもよい、スマホでLINEを見る人の手元のイメージ。顔は映さない。画面にはダミーの通知アイコン。スタイルはクリーンで説明的、派手なエフェクトは不要。
```

## B. 15 分動画用（横型・解説メイン）

### B-1. 章立て（目次のたたき台）

| 時間目安 | 内容 |
|----------|------|
| 0:00–1:00 | 自己紹介・この動画でできること（毎日の連絡をラクに） |
| 1:00–3:00 | 仕組みの図解：GAS → LINE API → 家族のスマホ |
| 3:00–6:00 | 事前準備：Google アカウント、LINE Developers のアカウント |
| 6:00–12:00 | LINE 側：チャネル作成、トークン発行、友だち追加（画面はモザイク前提で説明） |
| 12:00–14:00 | GAS 側：プロジェクト作成、スクリプトのプロパティにトークンを保存する理由 |
| 14:00–15:00 | デモ（ダミー）、GitHub 公開時の注意、まとめ、Note/README 誘導 |

※ 実際のクリック操作はバージョンで UI が変わるため、「公式ヘルプの検索ワード」を口頭で添えると長く使える動画になります。

### B-2. 詳細台本プロンプト（生成 AI 用）

```
あなたは日本語の教育系YouTuber向け台本ライターです。初心者向けに、次のトピックで15分の動画台本を書いてください。

トピック:
- 家族への「帰宅時間」「食事の有無」などの連絡を、Google Apps Script と LINE Messaging API で送る
- 個人利用・無料枠で足りることが多いことに触れる

要件:
- 視聴者はプログラム未経験。用語は初出でかんたんに定義
- 「コードの一行一行」より「何をどのサービスでするか」を優先。コードは短い例か省略可
- セキュリティ: トークンをGitHubや動画に載せない、スクリプトのプロパティに入れる、漏れたら再発行
- 最後に: GitHubのREADMEとNote記事への誘導文

出力:
1. チャプター分けされた台本（ナレーション全文、各チャプターに想定秒数）
2. 画面に出す図解の指示（テキストで簡潔に）
3. 概要欄テンプレート（リンク用プレースホルダ付き）
4. 初心者がつまずきやすいポイントTOP5と一言ヒント
```

### B-3. サムネイル文案プロンプト（任意）

```
YouTubeサムネイル用。テキストは大きく最大8文字×2行まで。内容は「帰宅連絡」「LINE自動」。背景は明るくシンプル。人物写真は使わない。日本語。
```

## C. 編集時チェックリスト

- [ ] トークン・シークレット・実 ID が 1 フレームも映っていない  
- [ ] エラー画面に個人情報が写っていない  
- [ ] 「無料」と言う箇所は「個人利用では多くの場合」など断定を弱めると安全  
- [ ] 概要欄に公式ドキュメント（LINE Messaging API、Apps Script）へのリンク

---

必要に応じて、実際のリポジトリ URL や Note の URL が決まったら、プロンプト内の `プレースホルダ` を差し替えてください。

---

## D. 撮影・投稿の流れ（初めての方向け）

画面操作の詳細は **[setup-beginner.md](setup-beginner.md)** の「F. note」「G. YouTube」を参照してください。ここでは流れだけ書きます。

1. **台本**：このファイルの A・B のプロンプトで AI に台本を出してもらい、自分用に直す。  
2. **撮影**：スマホまたは PC の画面録画。トークン・ユーザー ID は映さない。  
3. **編集**：不要な部分をカットし、概要欄用の文を書く。  
4. **YouTube にアップロード**：説明欄に GitHub と note の URL を貼る。  
5. **note に記事**：動画の補足として手順書へのリンクを貼ると、読者が迷いにくいです。
