# はじめての方向け：LINE・GAS・GitHub・Note・YouTube 手順書

このドキュメントは、**パソコンや Web サービスの操作に不慣れな方**でも、画面の流れを追えるように書いています。  
用語は初めて出るときだけ短く説明します。

---

## このプロジェクトでやることの全体像

1. **LINE** で「メッセージを送るための鍵（トークン）」と「公式アカウント」を用意する。  
2. **Google スプレッドシート**に、今日の「帰宅予定・食事・メモ」を書く欄を作る（またはプロパティだけでテスト）。  
3. **Google Apps Script（GAS）**にこのリポジトリのコードを貼り、秘密情報を「スクリプトのプロパティ」に保存する。  
4. **テスト送信**が届けば成功。必要なら **毎日決まった時間に送る**設定をする。  
5. 余裕があれば **GitHub** にコードを公開し、**Note** に記事、**YouTube** に動画を載せる。

順番は 1 → 2 → 3 → 4 が先です。5 はあとからで大丈夫です。

## A. LINE Developers（Messaging API）の準備

### A-1. アカウント

1. ブラウザで [LINE Developers](https://developers.line.biz/ja/) を開く。  
2. LINE アカウントでログインする。

### A-2. プロバイダーとチャネル

1. コンソールで **プロバイダー** を作成（名前は自由。例：自宅用）。  
2. **新しいチャネルを作成** → 種類は **Messaging API** を選ぶ。  
3. 表示名・アイコンなどは画面の指示に従う（後から変更も可能）。

### A-3. チャネルアクセストークン（長期）

1. 作成したチャネルの **Messaging API** タブを開く。  
2. **チャネルアクセストークン（長期）** を発行し、**コピー**する。  
3. この文字列を **メールやチャットに貼らず**、あとで GAS の **スクリプトのプロパティ** `LINE_CHANNEL_ACCESS_TOKEN` にだけ貼る（メモ帳に一時保存する場合は、共有 PC では注意）。

### A-4. 友だち追加とユーザー ID

通知を受け取る家族のスマホで、あなたの **Messaging API の公式アカウント**を **友だち追加**する（QR コードはチャネル設定画面にあります）。

**ユーザー ID（`U` で始まる文字列）** が必要です。次のどちらかで取得できます。

- **方法 1（推奨・このリポジトリの Webhook）**  
  後述の **「ウェブアプリとしてデプロイ」** まで進め、LINE Developers の **Webhook URL** にその URL を設定する。  
  友だち追加やメッセージを一度送ると、GAS の **実行ログ**に userId が出るか、空なら `LINE_TO_USER_ID` に自動保存されます。

- **方法 2**  
  LINE の公式ドキュメントや、Messaging API のデバッグ用ツールで userId を確認する（UI は変更されることがあります）。

### A-5. Webhook の利用（任意だが userId 取得に便利）

1. Messaging API の設定で **Webhook の利用** をオンにする。  
2. **Webhook URL** には、後で GAS から発行する **ウェブアプリの URL** を貼る（次のセクション）。  
3. **チャネルシークレット** は「署名検証」用にスクリプトのプロパティ `LINE_CHANNEL_SECRET` に保存すると安全です（未設定でも動くが、検証はスキップされます）。

## B. Google スプレッドシート

1. [Google スプレッドシート](https://sheets.google.com/) で **新規**を作成。  
2. 1 枚目のシートで、次のように入力（詳細は [spreadsheet-layout.md](spreadsheet-layout.md)）。  
   - **B1**：帰宅予定（例：`19:30頃`）  
   - **B2**：食事（例：`あり`）  
   - **B3**：メモ（任意）  
3. **このファイルから**「拡張機能 → Apps Script」を開くと、スクリプトとシートが紐づき、`SPREADSHEET_ID` は不要です。

---

## C. Google Apps Script（コードの入れ方）

### C-1. エディタを開く

- **スプレッドシートに紐づける場合**：そのファイルで「拡張機能 → Apps Script」。  
- **紐づけない場合**：[script.google.com](https://script.google.com/) で「新しいプロジェクト」。その場合は `SPREADSHEET_ID` をプロパティに設定。

### C-2. ファイルをコピーする

1. このリポジトリの `src/Code.gs` の中身をすべてコピーし、GAS の `コード.gs`（名前は変更可）に貼り付ける。  
2. 同様に `src/Webhook.gs` を **別のファイル**（例：`Webhook.gs`）として追加する。  
   - 左の「+」で「スクリプト」を追加し、ファイル名を `Webhook` にする。  
3. **マニフェスト（appsscript.json）** を反映する：左の **歯車 → プロジェクトの設定** で、「`appsscript.json` マニフェストをエディタで表示する」を **オン** にする。エディタに出た `appsscript.json` を、このリポジトリの `src/appsscript.json` の内容に置き換える（外部 URL へのアクセスとスプレッドシートの権限が含まれます）。  
   - 反映後、一度保存してから `sendTestMessage` を実行すると、権限の承認画面が出やすくなります。

### C-3. スクリプトのプロパティ

1. 左の **歯車（プロジェクトの設定）** → **スクリプトのプロパティ**。  
2. [script-properties.md](script-properties.md) の表に従い、`LINE_CHANNEL_ACCESS_TOKEN` などを追加。

### C-4. 初回の権限承認

1. エディタで関数 `sendTestMessage` を選び、**実行**をクリック。  
2. **権限を確認** → Google アカウントを選び、**詳細 →（安全でないページへ）→ 許可** など画面に従う（初回のみ）。  
3. **実行ログ**を開き、エラーがないか確認。  
4. 家族の LINE にテスト文が届けば成功です。

届かない場合は、トークン・`LINE_TO_USER_ID`・友だち追加を再確認してください。

### C-5. 本番の送信

関数 `sendFamilyUpdate` を実行すると、スプレッドシート（またはフォールバック）の内容で通知が送られます。

### C-6. 時間トリガー（毎日自動）

1. 左の時計アイコン **トリガー** → **トリガーを追加**。  
2. 実行する関数：`sendFamilyUpdate`  
3. イベントのソース：**時間主導型**  
4. 時刻の種類：**日タイマー** など、希望の時刻を選ぶ。

## D. Webhook 用ウェブアプリのデプロイ（ユーザー ID 取得・署名検証）

1. GAS エディタ右上 **デプロイ** → **新しいデプロイ** → 種類 **ウェブアプリ**。  
2. 次のように設定する例：  
   - **説明**：任意（例：`v1`）  
   - **次のユーザーとして実行**：自分  
   - **アクセスできるユーザー**：**全員**（LINE のサーバーから呼ばれるため）  
3. **デプロイ**後に表示される **URL** をコピーする。  
4. LINE Developers の **Webhook URL** にその URL を貼る（末尾に `/exec` が付く形式になっていることが多いです。表示された URL をそのまま使ってください）。  
5. **接続確認**や **Webhook の再送**があれば、LINE の画面から実行。

ブラウザでその URL を開き、`OK (LINE Webhook endpoint)` のような表示が出れば `doGet` は動いています。

## E. GitHub（はじめて使う方向け）

### E-1. GitHub アカウント

1. [GitHub](https://github.com/) で **Sign up**（無料で可）。  
2. メール認証まで完了する。

### E-2. リポジトリを新規作成

1. ログイン後、右上 **+** → **New repository**。  
2. **Repository name** に名前（例：`line-family-notify-gas`）。  
3. **Public** を選ぶ（公開用）。  
4. **Create repository**。

### E-3. パソコンからアップロード（Git をまだ使わない方法）

1. 作成した空のリポジトリのページに、**uploading an existing file** のようなリンクがある場合、そこからファイルをドラッグ＆ドロップできる。  
2. または **GitHub Desktop**（公式アプリ）を入れ、**File → Add local repository** でこのフォルダを指定し、**Publish**（初回はログインとリポジトリ名の指定が必要）。

**絶対に含めないもの**：トークン、チャネルシークレット、実データ入りのメモ。`.gitignore` で `.clasp.json` を除外している場合、そのファイルはアップされません。

### E-4. 誤って秘密を載せた場合

- すぐに LINE でトークン再発行。  
- GitHub の [公式ヘルプ](https://docs.github.com/ja) で「機密情報の削除」を検索し、履歴の扱いを確認する。

## F. note（はじめて書く方向け）

1. [note](https://note.com/) でアカウント作成（メールや SNS 連携）。  
2. **記事を書く** → タイトルと本文を入力。  
3. このリポジトリの `docs/note-article.md` を参考に、**自分の言葉**に直して貼る。  
4. **GitHub の公開 URL** が決まったら、記事内の「コードは GitHub で」とある部分に **リンクを貼る**。  
5. **公開**を押す前に、誤ってトークンを書いていないか確認する。

無料公開で問題ありません。画像を載ける場合も、QR やトークンが写っていないか確認してください。

---

## G. YouTube（はじめて投稿する方向け）

1. Google アカウントで [YouTube](https://www.youtube.com/) にログイン。  
2. 右上のカメラアイコンから **動画をアップロード**（初回はチャンネル作成の案内あり）。  
3. 動画ファイルを選び、**タイトル・説明**を入力。  
4. 説明欄に **GitHub の URL** と **note の URL** を書く。  
5. **公開設定**（公開・非公開・限定公開）を選び、**公開**。

台本や構成は `docs/video-prompts.md` を利用できます。撮影では **トークンや ID が映らない**ようにしてください。
